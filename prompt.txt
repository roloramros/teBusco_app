## CONTEXTO DEL PROYECTO

API REST en Node.js con Express 5, ES Modules, PostgreSQL (cliente `pg`), y Firebase Admin 
para notificaciones push. Archivos relevantes:

- `src/controllers/authController.js` — controlador de autenticación (contiene `registro`)
- `src/services/notificationService.js` — exporta `sendNotification(...)`

### Firma de sendNotification (úsala exactamente así):
```js
await sendNotification({
  usuario_id,        // UUID del receptor
  actor_id,          // null si es el sistema quien notifica
  tipo,              // string del enum tipo_notificacion_v2
  titulo,            // string
  cuerpo,            // string
  datos_extra,       // objeto plano { clave: valor } — los valores se convierten a string internamente
  fcm_token          // string | null — el token FCM del dispositivo del receptor
})
```

`sendNotification` ya maneja internamente dos cosas a la vez:
1. Persistencia en la tabla `notificaciones` de la BD (el usuario la verá en su panel)
2. Envío del push via Firebase Cloud Messaging al dispositivo

No es necesario hacer ningún INSERT adicional en `notificaciones` ni llamar a Firebase directamente.

### Enum relevante en la BD:
El campo `tipo` debe ser uno de los valores del enum `tipo_notificacion_v2`. 
El valor apropiado para mensajes del sistema es `'sistema_alerta'`.

## FUNCIÓN A MODIFICAR

La función `registro` en `authController.js`. Ya hace:
1. Validaciones (username, email, teléfono únicos)
2. Hash de contraseña
3. INSERT en `usuarios`
4. Si es chofer, INSERT en `choferes`
5. Genera JWT y crea sesión
6. Hace `COMMIT` y retorna `response.created(...)` con el token y datos del usuario

## FEATURE A IMPLEMENTAR

Después del `COMMIT` y antes del `return response.created(...)`, añadir el envío de una 
notificación de bienvenida al nuevo usuario. La notificación debe:

- Usar `usuario_id` del usuario recién creado
- `actor_id`: null (es el sistema quien notifica, no otro usuario)
- `tipo`: `'sistema_alerta'`
- `titulo`: `'¡Bienvenido a Te Busco! 🚕'`
- `cuerpo`: personalizado según el tipo de usuario:
  - Si `tipo === 'pasajero'`: `'Tu cuenta está lista. ¡Publica tu primera solicitud de viaje y encuentra tu chofer!'`
  - Si `tipo === 'chofer'`: `'Tu cuenta está lista. ¡Empieza a responder solicitudes y gana dinero con Tu Vehículo!'`
  - Para cualquier otro tipo: `'Tu cuenta ha sido creada con éxito. ¡Bienvenido!'`
- `datos_extra`: `{}` (objeto vacío)
- `fcm_token`: usar el `fcm_token` que viene del body del request (ya está desestructurado 
  al inicio de la función como `const { ..., fcm_token } = req.body`). Puede ser null si 
  el usuario no lo envió — `sendNotification` lo maneja sin problema.

## REGLAS IMPORTANTES

- El import de `sendNotification` debe añadirse al inicio del archivo si no existe ya.
- El envío de la notificación va FUERA de la transacción (después del COMMIT), igual que 
  en `solicitudController.js`. Nunca dentro del bloque try/catch de la transacción, para 
  no hacer rollback si falla la notificación.
- Si `sendNotification` lanza un error, no debe romper el flujo — envuélvela en un 
  try/catch propio que solo haga `console.error(...)` y continúe.
- No modifiques ningún otro archivo ni ninguna otra función del proyecto.
