Eres un desarrollador Android senior. Voy a darte el contexto de un proyecto y necesito que implementes una feature específica.

## CONTEXTO DEL PROYECTO

App Android en Java (no Kotlin). Arquitectura: Activities + ViewModel + LiveData + Retrofit2 + Gson. ViewBinding habilitado. El proyecto sigue estos patrones:

- Los ViewModels extienden `AndroidViewModel` y usan `MutableLiveData`
- Las llamadas a la API se hacen con `Call<ApiResponse<T>>` y callbacks Retrofit estándar
- Los diálogos usan `DialogFragment` (ver `AddVehicleDialogFragment` como referencia de estilo)
- `SessionManager` gestiona el token JWT almacenado en `SharedPreferences`
- `RetrofitClient` construye la instancia de Retrofit con el token en el header

### Clases relevantes existentes:
- `MyRequestsActivity` — muestra la lista de solicitudes del pasajero
- `MyRequestsAdapter` — adapter del RecyclerView de solicitudes
- `MainViewModel` — ViewModel que ya tiene métodos de solicitudes
- `ApiService` — interfaz Retrofit con todos los endpoints
- `RideRequest` — modelo de solicitud (tiene campos: `id`, `estado`, `choferSeleccionadoId`, etc.)
- `ApiResponse<T>` — wrapper genérico: `{ boolean ok, String message, T data }`

## FEATURE A IMPLEMENTAR

### 1. Modelo de request
Crear la clase `FinalizeRideRequest` con los campos `estrellas` (int) y `comentario` (String, puede ser null).

### 2. ApiService
Añadir el endpoint:
```java
@POST("api/solicitudes/{id}/finalizar")
Call<ApiResponse<Object>> finalizarViaje(
    @Path("id") String solicitudId,
    @Body FinalizeRideRequest body
);
```

### 3. ViewModel
En `MainViewModel`, añadir:
- `MutableLiveData<String> finalizarViajeError`
- `MutableLiveData<Boolean> viajeFinalizadoExito`
- Método `finalizarViaje(String solicitudId, int estrellas, String comentario)` que llame al endpoint y actualice los LiveData correspondientes

### 4. Dialog de valoración (`RatingDialogFragment`)
Crear un `DialogFragment` con:
- Título: "¿Cómo estuvo el viaje?"
- Un `RatingBar` (estilo nativo Android, 5 estrellas, stepSize 1.0) para seleccionar la puntuación — no usar librerías externas
- Un `TextInputLayout` + `TextInputEditText` con hint "Deja un comentario (opcional)" y `inputType textMultiLine`, máximo 200 caracteres
- Botón "Enviar" deshabilitado por defecto, se habilita cuando `RatingBar` > 0
- Botón "Cancelar" que cierra el dialog
- Comunicación con la Activity via interfaz: `interface OnRatingSubmitListener { void onSubmit(int estrellas, String comentario); }`
- La Activity/Fragment que lo use debe implementar esa interfaz

### 5. MyRequestsAdapter
En el método que renderiza cada tarjeta de solicitud, añadir lógica para mostrar un botón "Finalizar viaje" únicamente cuando `rideRequest.getEstado().equals("en_proceso")`. El click en ese botón debe invocar un callback al Activity (usar el mismo patrón de listener que ya tiene el adapter para otros botones).

### 6. MyRequestsActivity
- Implementar `RatingDialogFragment.OnRatingSubmitListener`
- Al recibir el callback del adapter (botón "Finalizar viaje" presionado), guardar el `solicitudId` actual y mostrar `RatingDialogFragment`
- En `onSubmit`, llamar a `viewModel.finalizarViaje(...)` 
- Observar `viajeFinalizadoExito`: cuando sea `true`, mostrar un `Snackbar` con "¡Viaje finalizado! Gracias por tu valoración" y recargar la lista de solicitudes
- Observar `finalizarViajeError`: mostrar el mensaje de error en un `Snackbar`
